  return result.text;
}
🔬 Chapter 4 — Bug #3: The 300-Second Freeze
⑨ FROZEN!

The server responded but no news ever appeared. lastUpdated stayed null forever.

// Terminal log:
⏳ Groq 429 — waiting 298s then retrying
⏳ Groq 429 — waiting 298s then retrying
⏳ Groq 429 — waiting 298s then retrying
// ... × 6 articles = 30 min freeze!
⑩

Root Cause

1
Groq returns HTTP 429 with header:
Retry-After: 298
2
Old code: "OK, I'll wait 300 seconds!"
3
6 articles × 300s = 30 minute freeze
4
updateNews() never completes → lastUpdated = null → empty page
⑪

The Fix: Don't Wait — Fall Through Immediately

❌ Before: On 429, wait retry-after seconds (up to 300s!), then retry Groq
if (status === 429) {
  const wait = retryAfter * 1000;
  // 😱 blocks for 300s!
  await sleep(wait);
  return await retryGroq();
}
✅ After: On 429, throw immediately → fallback catches it → use Google Translate
// No retry logic at all.
// 429 throws like any error.
// Fallback catches it instantly.
const r = await axios.post(...)
return r.data.choices[0].message.content;
⚡ Result: updateNews() completes in ~5 seconds even when Groq is fully rate-limited!

Bug #5: Wrong Language Code

Fix for Both: Parallel Processing + Correct Code

❌ Before: for...await (serial)

}
✅ After: Promise.allSettled() (parallel)
await Promise.allSettled(

    summarize(articles, lang)
  )
);

⑲

🔁 方案 A

Groq 指数退避重试

Groq 超限或超时？别急着放弃，先等一等再试！

// 最多重试 2 次
attempt 1: fail → wait 3s → retry
attempt 2: fail → wait 6s → retry
attempt 3: fail → throw ↓ 进入 B
等待时间 = 3000 × 2^(attempt-1)
减少"偶发失败"变成"fallback"的概率
⑳

♻️ 方案 B

全批次结束后二次扫描

1
第一轮翻译完成后，收集所有 translationMethod === 'fallback' 的文章
2
等待 30 秒，让 Groq TPM 配额恢复
3
只对这几篇文章重新跑一遍翻译（再次走 A 的重试逻辑）
4
成功则替换原条目；仍失败则保持 fallback
㉑

🚫 方案 C

写入缓存前过滤

🛡️
没翻译完的文章，一条都不许进缓存！
const toCache = summarized

    a.translationMethod !== 'fallback'
  );

⚙️ Chapter 8 — The TypeScript Time Bomb & 22 Green Lights
㉓ DEPRECATED!

VS Code 用 TypeScript 6.x 语言服务分析 tsconfig.json，弹出两条红色警告：

⛔ Option 'target=ES5' is deprecated and will stop functioning in TypeScript 7.0.
⛔ Option 'moduleResolution=node10' is deprecated and will stop functioning in TypeScript 7.0.
为什么项目的 tsc 不报错，VS Code 却报错？
㉔

双管齐下修复

❌ 旧配置
"target": "es5"
"moduleResolution": "node"
typescript: "^4.9.5"
✅ 新配置
"target": "ES2017"
"moduleResolution": "bundler"
typescript: "^5.9.3"
!
"bundler" 是 TS 5.0 引入的选项，专为 webpack / CRA 等打包器设计。用旧 TS 4.x 会直接报错，必须升级。
!
VS Code 用自带的 TS 6.x 解析 tsconfig；项目用的是 node_modules 里的 TS。两套版本各自独立。
㉕
