English · Deutsch · 中文

What Happened

1
Article 1 → Groq call ✅ translated!
2
Article 2 → Groq call ✅ maybe ok…
3
Article 3 → 429 RATE LIMITED
4
Articles 4–6 → All failed silently
5
Result: 2/6 translated. Looked like English!
⑤

The Fix

✅ Add a 10-second delay between each article when using Groq.

10s gap × 6 articles = ~1,500 tokens/min
Well under the 6,000 TPM limit!
// Only delay when Groq succeeded
if (translationMethod === 'Groq') {
  await sleep(10000);
}
🔬 Chapter 3 — Bug #2: The Fallback That Never Worked
⑥ 🤦
Wait — we had a MyMemory fallback!
Why didn't it kick in?
⑦

The Hidden Limit

MyMemory
Free Tier
＝
5,000 chars
per DAY
One refresh cycle = 6 articles × 2 fields × 480 chars =
~5,760 chars — already over the limit on the very first run!
⑧

The Fix: Replace MyMemory with google-translate-api-x

❌ MyMemory (Before)

• 5,000 chars/day (anonymous)
• Needs account registration
• Exhausted on first refresh
• Website too broken to sign up!

✅ google-translate-api-x (After)

• No registration needed
• No hard daily limit
• Already in package.json!
• Supports zh-CN, de, and more

const translate = require('google-translate-api-x');

async function translateText(text, targetLang) {
  const result = await translate(text, { from: 'en', to: targetLang });
  return result.text;
}
🔬 Chapter 4 — Bug #3: The 300-Second Freeze
⑨ FROZEN!

The server responded but no news ever appeared. lastUpdated stayed null forever.

// Terminal log:
⏳ Groq 429 — waiting 298s then retrying
⏳ Groq 429 — waiting 298s then retrying
⏳ Groq 429 — waiting 298s then retrying
// ... × 6 articles = 30 min freeze!
⑩

Root Cause

1
Groq returns HTTP 429 with header:
Retry-After: 298
2
Old code: "OK, I'll wait 300 seconds!"
3
6 articles × 300s = 30 minute freeze
4
updateNews() never completes → lastUpdated = null → empty page
⑪

The Fix: Don't Wait — Fall Through Immediately

❌ Before: On 429, wait retry-after seconds (up to 300s!), then retry Groq
if (status === 429) {
  const wait = retryAfter * 1000;
  // 😱 blocks for 300s!
  await sleep(wait);
  return await retryGroq();
}
✅ After: On 429, throw immediately → fallback catches it → use Google Translate
// No retry logic at all.
// 429 throws like any error.
// Fallback catches it instantly.
const r = await axios.post(...)
return r.data.choices[0].message.content;
⚡ Result: updateNews() completes in ~5 seconds even when Groq is fully rate-limited!
🔬 Chapter 5 — Bug #4 & #5: The Sequential Queue & Wrong Language Code
⑫

Bug #4: Serial Processing

The server processed each language one after another:

EN starts
→
done ~5s ✅
DE starts after EN
→
done ~65s 😐
ZH starts after DE
→
done ~125s 😡
German and Chinese users waited 2 minutes longer than English users!
⑬

Bug #5: Wrong Language Code

🈲
Google Translate rejected 'zh':
"language zh unsupported"

Chinese (Simplified) requires the code zh-CN, not zh.

// Before:
zh: 'zh'

// After:
zh: 'zh-CN' // ✅
⑭

Fix for Both: Parallel Processing + Correct Code

❌ Before: for...await (serial)
for (const lang of ['en','de','zh']) {
  // waits for en → then de → then zh
  await summarize(articles, lang);
}
✅ After: Promise.allSettled() (parallel)
await Promise.allSettled(
  ['en','de','zh'].map(lang =>
    summarize(articles, lang)
  )
);
EN ↗
DE ↗
all start at the same time!
ZH ↗
→
✅ All done
~10s
