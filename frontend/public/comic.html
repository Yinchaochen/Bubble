🔒 Chapter 6 — Security & Performance
① PROTECTED!

🛡️ Security Measures

1
CORS: Only whitelisted domains can reach the backend — all others are rejected
2
Rate Limiting: Max 100 requests per 15 minutes to prevent abuse
3
Helmet: Automatically adds secure HTTP headers to prevent XSS and other attacks
4
.env file: API keys are never hardcoded — stored as environment variables

🧊 The Cold Start Problem

😴 Server Asleep

No visitors for a while → Vercel freezes the Serverless Function to save resources. The server is essentially off.

🚪 You Knock

You open the site → your request arrives at a frozen server. It must boot up from scratch before it can respond.

⏳ Booting…

The cold start takes several seconds. Your browser shows a spinner or times out while waiting.

🔥 Now it's Hot

After the first load, the function stays warm for a while. Subsequent visits feel instant — until it sleeps again.

🔧 Why Vercel does this

1
Vercel's free tier uses Serverless Functions — they only run when a request arrives
2
No traffic = server shuts down = zero hosting cost while idle
3
The trade-off: the first visitor after a quiet period pays the startup cost
💡 This is expected behaviour on the free tier — not a bug in the app!
